Project Overview

What is this project about?

This is a Sales Data Analysis & Visualization System — a complete end-to-end Python application that can read sales data from any CSV, Excel (.xlsx), or JSON file.

Once the data is loaded, the system automatically cleans it (removes duplicates, fixes missing values), engineers new features (like month, year, profit margin), and runs a full Exploratory Data Analysis (EDA).

Finally, it generates static PNG charts and launches a fully interactive Plotly Dash web dashboard — viewable in any browser at http://127.0.0.1:8050.

src/loader.py · Step 1 of 5

📂 Load Data

How the system detects and reads your sales file — CSV, Excel, or JSON — into a pandas DataFrame ready for the next step.

How loader.py Works Step by Step
Step 1.1 — Receive the File Path

The function accepts a file path string

The main entry point calls load_data(file_path) and passes in the path to the user's data file. This could be typed manually or passed as a command-line argument. At this point, no file has been opened yet — we only have the path string.

def load_data(file_path: str) -> pd.DataFrame:
    # file_path example: "data/sample_data.xlsx"
    path = Path(file_path)
Why this design?

Accepting a string argument makes the function reusable — the same loader works whether the path comes from a terminal argument, a user prompt, or a test case.

Step 1.2 — Detect the File Extension

Automatically choose the right reader

Python's built-in pathlib.Path is used to extract the file extension (e.g., .csv, .xlsx, .json). The extension is lowercased to handle uppercase variants like .CSV or .XLSX without extra logic.

ext = path.suffix.lower() # e.g. ".xlsx", ".csv", ".json"

if ext == ".csv":
    df = pd.read_csv(file_path)
elif ext in (".xlsx", ".xls"):
    df = pd.read_excel(file_path)
elif ext == ".json":
    df = pd.read_json(file_path)
Why not just always use read_csv?

Each file format has different internal structure. CSV is plain text with commas; XLSX is a compressed XML binary; JSON is nested key-value pairs. Pandas provides a dedicated reader for each that understands the format correctly.

Step 1.3 — Handle Unknown Formats

Raise a clear error for unsupported files

If the extension doesn't match any supported format, the function raises a ValueError with a helpful message. This prevents silent failures where the program might try to process garbage data without knowing why.

else:
    raise ValueError(f"Unsupported file type: {ext}")
Step 1.4 — Return the DataFrame

Pass a clean DataFrame to the next step

After loading, the function prints a brief confirmation showing how many rows and columns were loaded. Then it returns the DataFrame object, which is immediately passed to cleaning.py in the next pipeline step.

print(f"✅ Loaded {len(df)} rows × {len(df.columns)} columns")
return df

src/cleaning.py · Step 2 of 5

🧹 Clean Data

How the system removes dirty data — duplicates, nulls, and type mismatches — so every downstream step works on reliable, consistent data.

How cleaning.py Works Step by Step
Step 2.1 — Remove Duplicate Rows

Drop exact row duplicates

The first cleaning operation calls drop_duplicates() which scans every row and removes any that are completely identical to a previous row. The sample data intentionally has 5 injected duplicates — this step removes all of them.

before = len(df)
df = df.drop_duplicates()
print(f"Removed {before - len(df)} duplicate rows")
Why this matters

Duplicate rows skew aggregations — a sale counted twice doubles its contribution to totals and averages. Removing duplicates ensures accurate revenue figures.

Step 2.2 — Fill Numeric Missing Values

Replace NaN in numeric columns with the median

For each numeric column (like Sales, Profit, Quantity), missing values are filled with the column's median. The median is preferred over mean because it isn't distorted by outlier values — a single huge order won't inflate the fill value.

for col in df.select_dtypes(include='number').columns:
    df[col].fillna(df[col].median(), inplace=True)
Step 2.3 — Drop Remaining Null Rows

Remove rows where essential data is still missing

After filling numerics, any rows that still have null values in critical text columns (like Category or Region) are dropped entirely. These rows cannot contribute to analysis without category labels.

df = df.dropna(subset=['Sales'])
# Sales column is essential — drop rows without it
Step 2.4 — Reset the DataFrame Index

Reindex so row numbers are continuous again

After dropping rows, the index has gaps (e.g., 0, 1, 4, 7...). Resetting ensures a clean 0-to-N index, preventing potential off-by-one errors in later processing.

df = df.reset_index(drop=True)
print(f"✅ Cleaned DataFrame: {len(df)} rows remain")
return df

src/eda.py · Step 3 of 5

⚙️ Feature Engineering & EDA

How the system creates new analytical columns from raw data and then extracts statistical insights through Exploratory Data Analysis.

How eda.py Works Step by Step
Step 3.1 — Parse Dates

Convert Order Date column to datetime type

The Order Date column arrives as plain text (e.g., "2023-04-15"). pandas' to_datetime() converts it into a real datetime object, unlocking time-based operations like extracting month and year.

df['Order Date'] = pd.to_datetime(df['Order Date'], errors='coerce')
Step 3.2 — Extract Month & Year

Create new time-based feature columns

Once dates are parsed, two new columns are created: Order Month (1–12) and Order Year (2022, 2023, etc). These power the monthly trend charts and allow year-over-year comparison.

df['Order Month'] = df['Order Date'].dt.month
df['Order Year'] = df['Order Date'].dt.year
Why extract these?

Raw dates can't be grouped directly into "monthly totals". Extracting the month as a number lets groupby('Order Month') work instantly to produce a monthly sales series for trend charts.

Step 3.3 — Compute Profit Margin

Add a derived percentage column

A new column Profit Margin % is calculated by dividing Profit by Sales and multiplying by 100. This derived metric is more useful for comparison than raw profit values — it shows efficiency regardless of order size.

df['Profit Margin %'] = (df['Profit'] / df['Sales']) * 100
Step 3.4 — Summary Statistics

Print describe() output to terminal

The describe() method generates count, mean, std, min, 25th percentile, median, 75th percentile, and max for every numeric column. This is printed to the terminal as a quick sanity check of the data's range and distribution.

print(df.describe().to_string())
Step 3.5 — Group Aggregations

Sales totals by Category and Region

Two grouped summaries are computed and printed: total and average sales grouped by Category, and by Region. These inform which segments are most profitable and guide which charts to highlight in the dashboard.

category_sales = df.groupby('Category')['Sales'].agg(['sum', 'mean'])
region_sales = df.groupby('Region')['Sales'].agg(['sum', 'mean'])
Step 3.6 — Correlation Matrix

Find relationships between numeric columns

The corr() method computes the Pearson correlation coefficient between every pair of numeric columns. A value near +1 means strong positive relationship (higher sales → higher profit). This feeds the heatmap chart in visualization.py.

corr_matrix = df.select_dtypes(include='number').corr()
print(corr_matrix)

src/visualization.py · Step 4 of 5

📊 Static Charts

How matplotlib and seaborn are used to generate 6 publication-quality PNG charts saved to the charts/ folder.

How visualization.py Works Step by Step
Step 4.1 — Setup Plot Style

Apply consistent theme across all charts

Before drawing any chart, a global Seaborn theme is applied using sns.set_theme(). This sets a consistent background grid, font sizes, and color palette for all subsequent plots so they look cohesive as a set.

import matplotlib.pyplot as plt
import seaborn as sns

sns.set_theme(style="whitegrid", palette="muted")
Step 4.2 — Chart 1: Sales by Category (Bar)

Compare total revenue across product categories

Groups the DataFrame by Category, sums the Sales column, and plots a vertical bar chart. Each bar represents one category (e.g., Technology, Furniture, Office Supplies). Saved as charts/sales_by_category.png.

cat_sales = df.groupby('Category')['Sales'].sum()
cat_sales.plot(kind='bar', color='steelblue')
plt.savefig('charts/sales_by_category.png', dpi=150, bbox_inches='tight')
Step 4.3 — Chart 2: Monthly Sales Trend (Line)

Visualize how sales change over time

Groups by Order Month and plots a line chart. The X axis shows months 1–12, and the Y axis shows total sales per month. This reveals seasonal patterns — e.g., a November spike for holiday shopping.

monthly = df.groupby('Order Month')['Sales'].sum()
monthly.plot(kind='line', marker='o')
plt.savefig('charts/monthly_trend.png', dpi=150)
Step 4.4 — Chart 3: Profit vs Sales Scatter

Find the relationship between revenue and profit

Each dot represents one order. The X position is its Sales value, the Y position is its Profit. A positive correlation (dots trending up-right) confirms that higher-sales orders also tend to generate more profit.

plt.scatter(df['Sales'], df['Profit'], alpha=0.4)
plt.savefig('charts/profit_vs_sales.png', dpi=150)
Step 4.5 — Chart 4: Correlation Heatmap (Seaborn)

Visualize all numeric relationships at once

Takes the correlation matrix computed in eda.py and renders it as a color-coded grid using sns.heatmap(). Warm colors = positive correlation, cool colors = negative. Annotations show exact coefficient values.

sns.heatmap(corr_matrix, annot=True, fmt=".2f", cmap="coolwarm")
plt.savefig('charts/correlation_heatmap.png', dpi=150)
Step 4.6 — Charts 5 & 6: Region & Segment

Regional bar chart + Customer segment pie chart

Two final charts: a horizontal bar chart of sales by Region, and a pie chart showing customer Segment split (Consumer, Corporate, Home Office). All charts are saved as separate PNG files in the charts/ folder.

# Regional bar
df.groupby('Region')['Sales'].sum().plot(kind='barh')
plt.savefig('charts/region_sales.png')

# Segment pie
df['Segment'].value_counts().plot(kind='pie', autopct='%1.1f%%')
plt.savefig('charts/segment_pie.png')

src/dashboard.py · Step 5 of 5

🚀 Interactive Dashboard

How Plotly Dash assembles an interactive web application with live filter controls, fully in Python — no HTML or JavaScript required.

How dashboard.py Works Step by Step
Step 5.1 — Initialize the Dash App

Create the application object with Bootstrap theme

A Dash app object is created and given the BOOTSTRAP external stylesheet from dash-bootstrap-components. This one line loads the full Bootstrap CSS framework into the web app, enabling responsive grid layouts.

import dash
import dash_bootstrap_components as dbc

app = dash.Dash(__name__, external_stylesheets=[dbc.themes.BOOTSTRAP])
Step 5.2 — Build the Layout

Define the page structure using Python components

The dashboard layout is described as nested Python objects — dbc.Container, dbc.Row, dbc.Col act exactly like Bootstrap's grid system. Each chart area is wrapped in a dcc.Graph component which renders Plotly charts.

app.layout = dbc.Container([
    dbc.Row([dbc.Col(html.H1("Sales Dashboard"))]),
    dbc.Row([
        dbc.Col(dcc.Dropdown(id='region-filter', ...), md=4),
        dbc.Col(dcc.Graph(id='sales-chart'), md=8),
    ])
])
Step 5.3 — Add Filter Dropdowns

Region and Segment filter controls

Two dcc.Dropdown components are added — one listing all unique Regions, one listing all Segments. When the user picks a value from either dropdown, Dash automatically triggers the callback function to redraw the charts with filtered data.

dcc.Dropdown(
    id='region-filter',
    options=[{'label': r, 'value': r} for r in df['Region'].unique()],
    placeholder='Filter by Region...',
    multi=True
)
Step 5.4 — Write the Callback

Reactive function that redraws charts on filter change

The @app.callback decorator links dropdown values to chart outputs. Every time a filter changes, Dash calls the decorated function automatically with the new filter values. The function filters the DataFrame and returns updated Plotly figure objects.

@app.callback(
    Output('sales-chart', 'figure'),
    Input('region-filter', 'value')
)
def update_chart(selected_regions):
    filtered = df[df['Region'].isin(selected_regions)] if selected_regions else df
    return px.bar(filtered, x='Category', y='Sales')
Why use callbacks?

Callbacks are what make Dash "reactive" — the page updates instantly without any page reload. It's like a spreadsheet formula: change an input, the output auto-recalculates. This is the core feature that makes Dash powerful for data dashboards.

Step 5.5 — Start the Server

Launch the local web server

The final line starts a Flask development server on port 8050. The terminal prints the local URL. Anyone on the same machine can open a browser and interact with the live dashboard. Press Ctrl+C to stop it.

if __name__ == '__main__':
    app.run_server(debug=True, port=8050)

# Terminal output:
# Dash is running on http://127.0.0.1:8050/

generate_sample_data.py · Demo Generator

🗂️ Demo Data Generator

How the script builds a realistic 300-row sales dataset from scratch using numpy, and deliberately injects errors to showcase the cleaning step.

How generate_sample_data.py Works Step by Step
Step G.1 — Define the Data Schema

Decide what columns the fake data will have

The script first defines lists of possible values for each categorical column — Regions, Categories, Segments, and a pool of 50 customer names. These pools are randomly sampled to create variety across the 300 generated rows.

regions = ['East', 'West', 'North', 'South']
categories = ['Technology', 'Furniture', 'Office Supplies']
segments = ['Consumer', 'Corporate', 'Home Office']
Step G.2 — Generate 300 Random Rows

Use numpy to create realistic numeric distributions

numpy's np.random functions generate realistic-looking numbers: Sales values follow a log-normal distribution (skewed toward smaller orders with occasional large ones), Quantities are integers 1–10, and Profit is derived from Sales with a random margin.

n = 300
sales = np.random.lognormal(mean=5, sigma=1, size=n).round(2)
quantity = np.random.randint(1, 10, size=n)
profit = (sales * np.random.uniform(0.05, 0.35, size=n)).round(2)
Why log-normal for sales?

Real sales data is rarely normally distributed — most orders are small, with occasional very large ones. A log-normal distribution mirrors this real-world pattern, making the demo data more representative.

Step G.3 — Inject 10 Null Values

Deliberately break some cells to demo cleaning

10 random cell positions in the Sales and Profit columns are set to np.nan. This ensures that when the user runs cleaning.py, they'll see the "filled 10 null values" message — proving the cleaning step actually works.

null_indices = np.random.choice(n, 10, replace=False)
df.loc[null_indices, 'Sales'] = np.nan
Step G.4 — Inject 5 Duplicate Rows

Add exact copies to prove deduplication works

5 rows from the DataFrame are randomly selected and appended as exact duplicates. This makes the DataFrame temporarily 305 rows. After cleaning.py runs drop_duplicates(), it will return to 300 rows — visibly demonstrating the function's effect.

dupes = df.sample(5, random_state=42)
df = pd.concat([df, dupes], ignore_index=True)
# Now 305 rows (300 original + 5 duplicates)
Step G.5 — Save to Excel

Write the DataFrame to data/sample_data.xlsx

The final DataFrame (with intentional flaws) is saved as an Excel file using pandas' to_excel() method, which internally uses the openpyxl library. The index=False parameter prevents pandas from writing a useless row-number column.

df.to_excel('data/sample_data.xlsx', index=False)
print("✅ sample_data.xlsx created", len(df), "rows")

pd.read_csv, pd.read_excel, pd.read_json

These three functions are the entry point to every data pipeline. They parse raw file bytes into a structured DataFrame — pandas' core data structure. A DataFrame is like a spreadsheet table in memory: rows are records, columns are fields, and you can query them with code.

drop_duplicates, fillna, dropna, reset_index

pandas provides all the tools needed to fix bad data in just a few method calls. drop_duplicates() compares all rows and removes copies. fillna(df[col].median()) replaces NaN with the column median. dropna() removes rows that remain incomplete after filling.

df = df.drop_duplicates()

to_datetime, dt.month, dt.year, column creation

pandas makes it trivial to derive new columns from existing ones. pd.to_datetime() converts strings to real dates. The .dt accessor then gives access to date parts like .dt.month. New columns are created by simple assignment, like adding a spreadsheet formula column.

df['Order Date'] = pd.to_datetime(df['Order Date'])

df['Profit Margin %'] = (df['Profit'] / df['Sales']) * 100

groupby, agg, value_counts, describe

groupby() is one of pandas' most powerful features — it splits data into groups by a column value, then applies a function to each group. .agg(['sum','mean']) computes multiple statistics at once. describe() produces a full statistical summary in one call.

How numpy is Used in This Project
Usage 1 — Generate Random Sales Values

np.random.lognormal for realistic sales distribution

Real sales data doesn't follow a bell curve — most transactions are small with occasional large ones. np.random.lognormal() produces exactly this skewed distribution. It generates 300 positive float values that look like authentic sales figures ranging from ~$20 to ~$10,000.

sales = np.random.lognormal(mean=5, sigma=1, size=300)
Usage 2 — Inject Missing Values (NaN)

np.nan as the standard "missing data" sentinel

np.nan is the universally accepted representation of a missing number in Python's data ecosystem. pandas understands np.nan natively — all cleaning functions like fillna() and dropna() target exactly this value.

df.loc[null_indices, 'Sales'] = np.nan
# pandas reads np.nan as "missing" — cleaned by fillna()
Usage 3 — Random Index Selection

np.random.choice to pick random positions

np.random.choice(n, 10, replace=False) selects 10 unique random row numbers from the range 0–299 without repeats. These become the positions where null values are injected, ensuring the nulls are scattered randomly through the dataset.

null_indices = np.random.choice(300, 10, replace=False)
# Returns e.g. [42, 7, 189, 253, ...]
Usage 4 — Why numpy Matters Under the Hood

pandas is built on top of numpy arrays

Every pandas DataFrame column is secretly a numpy array. When pandas computes df['Sales'].median(), it delegates that math to numpy's optimized C code. This is why pandas is so fast — it uses numpy's compiled numerical routines rather than slower Python loops.

Performance context

numpy can compute the mean of 1 million numbers roughly 100× faster than a plain Python for loop, because it executes the calculation in compiled C/Fortran code rather than interpreted Python bytecode.

Usage 2 — Correlation Heatmap

sns.heatmap — color-coded correlation matrix

sns.heatmap() is the star of seaborn in this project. It takes the correlation matrix (a pandas DataFrame of numbers between -1 and +1) and renders it as a color-coded grid. Warm red = positive correlation, cool blue = negative, white = no relationship. Numbers are annotated in each cell.

corr = df.select_dtypes(include='number').corr()

    corr,
    annot=True, # show numbers inside cells

Usage 1 — Why You Need It Even Without Calling It

pandas delegates Excel I/O to openpyxl

When you call pd.read_excel('file.xlsx'), pandas does not have its own Excel parser built in. Instead, it checks if openpyxl is installed and uses it as the "engine" to decode the .xlsx binary format. If openpyxl is not installed, pd.read_excel() will throw a ModuleNotFoundError and refuse to run.

# This line requires openpyxl to be installed:
df = pd.read_excel('data/sample_data.xlsx')

# If openpyxl is missing, you'll see:
# ModuleNotFoundError: Missing optional dependency 'openpyxl'
Usage 2 — Writing Excel Files (to_excel)

generate_sample_data.py saves via openpyxl too

df.to_excel() in generate_sample_data.py also uses openpyxl internally — this time for writing. openpyxl constructs the .xlsx file format (which is actually a ZIP archive containing XML files) and writes it to disk. The result is a proper Excel file that opens in Microsoft Excel, Google Sheets, or LibreOffice.

# Also uses openpyxl under the hood:
df.to_excel('data/sample_data.xlsx', index=False)

# .xlsx is actually a ZIP of XML files:
# xl/worksheets/sheet1.xml ← the actual cell data
Usage 3 — The Dependency Chain

Understanding why it's in requirements.txt

openpyxl is an optional dependency of pandas — pandas doesn't install it automatically because not everyone needs Excel support. That's why it must be explicitly listed in requirements.txt. When you run pip install -r requirements.txt, both pandas and openpyxl are installed, ensuring the full Excel read/write pipeline works.

# requirements.txt
pandas
numpy
matplotlib
seaborn
plotly
dash
dash-bootstrap-components
openpyxl # ← required for pandas Excel support
Key insight

openpyxl is the only library in this project that you never import directly in your code — but it is absolutely essential. Forgetting to include it in requirements.txt is one of the most common deployment bugs for projects that use Excel files.
